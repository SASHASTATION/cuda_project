# CUDA optimizer refactor

Что изменено:

1. После каждого обновления весов метрики считаются заново на новых весах.
2. После каждого запуска ядра есть `CUDA_KERNEL_CHECK()`, чтобы ошибка не превращалась в нули.
3. Общая логика вынесена в `common.cuh`, а каждый stage остался в своём файле.
4. `benchmark.cu` только запускает стадии и пишет CSV.

Главная идея упрощения:
- CUDA оставляем только там, где реально есть параллельная работа.
- Управление обучением, логирование и сводная таблица — обычный C++ на CPU.

Сборка:

```bash
make benchmark ARCH=61
./benchmark
```
